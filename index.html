useSonarQube + Pytest generates Coverage
· 2021-03-20 · # sonarqube # pytest # coverage # python

Backgroud

 There is a DevOps practices plan processing in our company. I can use JIRA, Bitbucket, CI Pipeline and SonarQube. So I would like to use the code quality tool SonarQube in my local to explore. And I find there is a problem the Converage index of the sonar-scanner result is always 0% when I have generated the pytest scan report.

Project Structure

-project
    |-app
    |-tests
    |-coveragte.xml
    |-sonar-project.properties

Define a sonar-project.properties under the root project folder

sonar-project.properties

#----- Default SonarQube server
sonar.projectKey=XXX
sonar.host.url=XXX
sonar.login=XXX

#----- Default source code encoding
sonar.language=python
sonar.projectVersion=1.0
sonar.sources=app
sonar.sourceEncoding=UTF-8
sonar.dynamicAnalysis=reuseReports
sonar.core.codeCoveragePlugin=cobertura
sonar.python.coverage.reportPaths=coverage.xml

Install a related python package, run pytest and sonar-scanner

pip install pytest-cov
cd project
pytest --cov=app tests/ --cov-report=xml
sonar-scanner

Check info from python interpreter terminal

Check the result from Quality Gate in the SonarQube dashboard

......
INFO: Sensor Cobertua Sensor for Python coverage [python]
INFO: Python test coverage
INFO: Parsing report '...../app/tests/....'
......

Done
